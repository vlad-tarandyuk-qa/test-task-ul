# Чек-лист тестирования бизнес-процесса ЭТрН

## 1. Создание и заполнение первого титула

- [ ] Грузоотправитель может создать новую ЭТрН.
- [ ] Форма первого титула содержит обязательные поля: условия договора, сведения о сторонах сделки (грузоотправитель, перевозчик, грузополучатель), данные о водителе, описание груза, сопроводительные документы.
- [ ] При незаполнении любого обязательного поля кнопка «Подписать и отправить» неактивна или выводится ошибка.
- [ ] Можно сохранить черновик без подписания.
- [ ] После заполнения всех обязательных полей можно подписать и отправить первый титул.
- [ ] После отправки система устанавливает статус ЭТрН «Ожидает отгрузки со склада».

## 2. Валидация полей (по XSD)

- [ ] Поле «ИНН грузоотправителя» принимает только 10 или 12 цифр.
- [ ] Поле «Дата составления» не может быть позже текущей даты.
- [ ] Поле «Дата погрузки» не может быть раньше даты составления.
- [ ] Поле «Вес груза» не может быть отрицательным или нулевым (если бизнес-правило требует >0).
- [ ] Поле «Количество мест» – только целое положительное число.
- [ ] Поле «Номер ЭТрН» уникально в пределах грузоотправителя.
- [ ] Все коды (ОКПО, ОКСМ, БИК, коды видов груза) соответствуют справочникам.
- [ ] Поле `ДатаТрН` (дата составления) соответствует формату `ДД.ММ.ГГГГ`.
- [ ] Поле `ВрИнфГО` (время формирования) соответствует формату `ЧЧ:ММ:СС`.
- [ ] Атрибут `КНД` для грузоотправителя имеет фиксированное значение `1110430` (проверка при формировании XML).
- [ ] Поле `НомерТрН` не превышает 1000 символов (ограничение XSD).
- [ ] Поле `СпосПодтПолном` принимает только значения 1,2,3,4,5,6 (enum).
- [ ] При `СпосПодтПолном` = 3 проверяется наличие элемента `СвДоверЭл` с `НомДовер`, `ДатаВыдДовер`, `ИдСистХран`.
- [ ] При `СпосПодтПолном` = 5 проверяется наличие элемента `СвДоверБум` с `ДатаВыдДовер`, `ВнНомДовер`.
- [ ] Поле `УИД_ТрН` имеет длину 1–36 символов и уникально в системе.

## 3. Второй титул – погрузка

- [ ] Перевозчик получает первый титул, подтверждает получение.
- [ ] Водитель прибывает на погрузку, передаёт данные о погрузке.
- [ ] Если есть замечания при погрузке (несоответствие груза, повреждения, недостача) – активируется чекбокс «Замечания при погрузке», вносятся пояснения.
- [ ] Заполняются поля второго титула (фактические дата/время погрузки, масса, количество мест, состояние груза).
- [ ] Второй титул подписывается перевозчиком и отправляется.
- [ ] Система меняет статус на «Груз принят к перевозке».
- [ ] Генерируется QR-код для водителя, рассылается второй титул грузоотправителю и грузополучателю.

## 4. Третий титул – приёмка груза

- [ ] Водитель прибывает на место разгрузки, сообщает о прибытии.
- [ ] Грузополучатель принимает транспорт на разгрузку.
- [ ] Если груз принят – заполняются данные о приёмке (фактическая дата, состояние груза, замечания).
- [ ] Если груз не принят (отказ) – фиксируется причина, статус остаётся или идёт в спор.
- [ ] Третий титул подписывается грузополучателем и отправляется.
- [ ] Статус ЭТрН меняется на «Груз принят грузополучателем».
- [ ] Третий титул рассылается грузоотправителю и перевозчику.

## 5. Четвёртый титул – замечания при выгрузке (при наличии)

- [ ] Если при выгрузке обнаружены замечания (порча, недостача, излишки) – активируется чекбокс «Замечания при выгрузке».
- [ ] Вносятся данные о несоответствиях.
- [ ] Четвёртый титул подписывается перевозчиком и отправляется.
- [ ] Статус ЭТрН остаётся «Завершен» (или другой финальный статус) после отправки четвёртого титула.
- [ ] Четвёртый титул рассылается грузоотправителю и грузополучателю.

## 6. Завершение процесса

- [ ] После успешного получения груза и отсутствия замечаний статус ЭТрН становится «Завершен».
- [ ] При наличии четвёртого титула – статус также «Завершен».
- [ ] Все титулы (1,2,3,4) сохраняются и доступны для просмотра участниками.
- [ ] Завершённую ЭТрН нельзя редактировать.

## 7. Негативные сценарии и обработка ошибок

- [ ] Попытка подписать и отправить первый титул без обязательных полей – ошибка, отправка блокируется.
- [ ] Попытка подписать второй титул, если статус не «Ожидает отгрузки со склада» – ошибка.
- [ ] Отправка данных о погрузке без фактической погрузки (логический контроль) – предупреждение.
- [ ] Отказ грузополучателя от груза – корректный переход в статус «Отказ» или возможность аннулирования.
- [ ] При сбое сети во время подписания – данные не теряются, возможен повтор.
- [ ] Ввод специальных символов и скриптов в текстовые поля – экранирование, XSS-защита.
- [ ] Превышение максимальной длины текстовых полей (например, 255 символов для названия порта) – ошибка.

## 8. Интеграция и подписи

- [ ] Электронная подпись (УКЭП) корректно прикладывается при отправке каждого титула.
- [ ] Проверка подписи получателем проходит успешно.
- [ ] При использовании доверенности (СпосПодтПолном=3 или 5) – реквизиты доверенности заполнены и соответствуют требованиям.
- [ ] QR-код генерируется и содержит ссылку или идентификатор ЭТрН.
- [ ] Данные о маршруте, судне (название, тип, капитан) передаются корректно (если применимо).

## 9. Проверки ролей и прав доступа

- [ ] Грузоотправитель может создавать, редактировать и подписывать только первый титул.
- [ ] Перевозчик не может редактировать первый титул после подписания грузоотправителем.
- [ ] Перевозчик может создавать, редактировать и подписывать второй и четвёртый титулы.
- [ ] Грузополучатель может создавать, редактировать и подписывать только третий титул.
- [ ] Водитель (отдельная роль) не имеет права подписывать документы от имени перевозчика или грузополучателя, только передаёт данные о погрузке/разгрузке.
- [ ] Сторонний пользователь (не участвующий в перевозке) не видит ЭТрН в поиске и не может открыть её по прямой ссылке.
- [ ] После завершения перевозки ни одна из ролей не может редактировать документ (кроме, возможно, администратора).

## 10. Негативные переходы статусов

- [ ] Попытка создать второй титул (погрузка), когда статус не «Ожидает отгрузки со склада» – ошибка.
- [ ] Попытка создать третий титул (приёмка груза), когда статус не «Груз принят к перевозке» – ошибка.
- [ ] Попытка создать четвёртый титул (замечания при выгрузке) без активации чекбокса замечаний – ошибка или кнопка неактивна.
- [ ] Попытка повторно подписать уже подписанный титул – система блокирует или выводит предупреждение.
- [ ] Попытка отредактировать подписанный первый титул – изменения запрещены (кроме, возможно, через механизм исправлений `ИспрТрН`).
- [ ] Попытка удалить ЭТрН после её завершения – запрещено (или только с правами администратора).
- [ ] Переход из статуса «Завершен» обратно в любой предыдущий статус – невозможен.
