# Чек-лист тестирования бизнес-процесса ЭТрН

## 1. Создание и заполнение первого титула
- [ ] Грузоотправитель создаёт новую ЭТрН.
- [ ] Форма содержит обязательные поля: условия договора, стороны, водитель, груз.
- [ ] Без заполнения обязательных полей нельзя подписать и отправить.
- [ ] Можно сохранить черновик.
- [ ] После отправки статус «Ожидает отгрузки со склада».

## 2. Валидация полей (по XSD)
- [ ] ИНН грузоотправителя – 10 или 12 цифр.
- [ ] Дата погрузки не раньше даты составления.
- [ ] Вес груза >0, не отрицательный.
- [ ] Количество мест – целое положительное.
- [ ] Номер ЭТрН уникален.

## 3. Второй титул (погрузка)
- [ ] Перевозчик подтверждает получение.
- [ ] Водитель передаёт данные о погрузке.
- [ ] При замечаниях – активируется чекбокс, вносятся пояснения.
- [ ] Второй титул подписан → статус «Груз принят к перевозке».
- [ ] Генерируется QR-код.

## 4. Третий титул (приёмка груза)
- [ ] Водитель прибывает на разгрузку.
- [ ] Грузополучатель принимает груз или отказывается.
- [ ] При приёме – подписывается третий титул → статус «Груз принят грузополучателем».

## 5. Четвёртый титул (замечания при выгрузке)
- [ ] При замечаниях – активируется чекбокс, вносятся данные.
- [ ] Четвёртый титул подписан → статус «Завершен».

## 6. Завершение
- [ ] После приёмки без замечаний – статус «Завершен».
- [ ] Завершённую ЭТрН нельзя редактировать.

## 7. Негативные сценарии
- [ ] Подписание без обязательных полей – ошибка.
- [ ] Некорректный ИНН – ошибка.
- [ ] Ввод скриптов – XSS-защита.
- [ ] Превышение длины поля – ошибка.
- [ ] Отсутствие ЭП – ошибка.
- [ ] Дубликат номера ЭТрН – ошибка.

## 8. Подписи и интеграция
- [ ] УКЭП прикладывается корректно.
- [ ] При доверенности – реквизиты заполнены.
- [ ] QR-код содержит ссылку на ЭТрН.
