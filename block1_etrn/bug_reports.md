# Баг-репорты по результатам тестирования ЭТрН

## Баг-репорт №1 (CRITICAL)

**Заголовок:** Отсутствует проверка уникальности номера ЭТрН при создании – возможен дубль

**Предусловия:**  
- Пользователь (грузоотправитель) авторизован в системе.  
- Открыта форма создания новой ЭТрН (первый титул).

**Шаги воспроизведения:**
1. Создать новую ЭТрН, заполнить все обязательные поля.  
2. В поле «Номер ЭТрН» вручную ввести значение, которое уже существует в системе для этого грузоотправителя (например, `123-А`).  
3. Сохранить черновик.  
4. Подписать и отправить первый титул.

**Фактический результат:**  
Система позволяет создать и отправить ЭТрН с уже существующим номером. Два документа имеют одинаковый номер.

**Ожидаемый результат:**  
В соответствии с логикой документа и XSD (поле `НомерТрН` должно быть уникальным в пределах грузоотправителя), система должна выдать сообщение: «Электронная транспортная накладная с таким номером уже существует. Укажите другой номер».

**Серьёзность:** Critical  
**Приоритет:** High  
**Окружение:** Веб-интерфейс, версия 1.0  
**Приложение:** Скриншот с дублирующимся номером.

## Баг-репорт №2 (HIGH)

**Заголовок:** Некорректная валидация кода КНД при формировании файла обмена грузоотправителя

**Предусловия:**  
- Грузоотправитель создаёт ЭТрН (первый титул).  
- В XML-файле обмена формируется атрибут `КНД`.

**Шаги воспроизведения:**
1. Заполнить форму ЭТрН всеми обязательными данными.  
2. Через инструменты разработчика (или прямую правку XML) изменить значение атрибута `КНД` в исходящем файле на `1110431` (код для перевозчика) вместо правильного `1110430`.  
3. Подписать и отправить файл.

**Фактический результат:**  
Система принимает файл с неверным `КНД`. Документ ошибочно идентифицируется как информация перевозчика.

**Ожидаемый результат:**  
Согласно XSD (`КНД` для грузоотправителя должен иметь фиксированное значение `1110430`), система должна отклонить файл с ошибкой: «Неверный код формы документа (КНД). Ожидается 1110430».

**Серьёзность:** High  
**Приоритет:** Medium  
**Окружение:** Тестовый стенд, XML-шлюз  
**Примечание:** Нарушает целостность классификации документов, может привести к неверной маршрутизации.
