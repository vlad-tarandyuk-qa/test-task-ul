# Негативные проверки бизнес-процесса ЭТрН

| № | Негативный сценарий | Ожидаемое поведение |
|---|----------------------|----------------------|
| 1 | Попытка подписать и отправить первый титул без заполнения хотя бы одного обязательного поля (например, ИНН грузоотправителя) | Система блокирует отправку, подсвечивает незаполненное поле, выдаёт сообщение «Обязательное поле не заполнено». |
| 2 | В поле «ИНН грузоотправителя» ввести 11 цифр (неверная длина) | Ошибка: «ИНН должен содержать 10 или 12 цифр». |
| 3 | В поле «Дата погрузки» указать дату раньше даты составления ЭТрН | Ошибка: «Дата погрузки не может быть раньше даты составления документа». |
| 4 | В поле «Вес груза» указать 0 (ноль) | Ошибка: «Вес груза должен быть больше 0» (если бизнес-правило требует >0); иначе принимается. |
| 5 | В поле «Вес груза» указать отрицательное число (-100) | Ошибка: «Значение не может быть отрицательным». |
| 6 | В поле «Объем груза» указать отрицательное число (-1) | Ошибка: «Объём не может быть отрицательным». |
| 7 | В поле «Количество мест» указать дробное число (2,5) | Ошибка: «Количество мест должно быть целым числом». |
| 8 | Попытка подписать второй титул (погрузка), когда статус не «Ожидает отгрузки со склада» | Система отклоняет: «Недопустимый статус для выполнения операции. Сначала завершите первый титул». |
| 9 | Водитель передаёт данные о погрузке, но фактическая погрузка не производилась (логический контроль) | Система блокирует отправку, если не указаны фактические дата/время и состояние груза, либо выдаёт предупреждение: «Подтвердите, что погрузка выполнена». |
| 10 | Грузополучатель отказывается от груза (третий титул) без указания причины | Ошибка: «Укажите причину отказа от груза». |
| 11 | Попытка подписать четвёртый титул (замечания при выгрузке) без активации чекбокса «Замечания при выгрузке» | Кнопка «Подписать и отправить» неактивна или выводится ошибка: «Нет замечаний для заполнения четвёртого титула». |
| 12 | Ввод специальных символов и скриптов в текстовые поля (например, `<script>alert(1)</script>` в поле «Особые заявления и отметки») | Система экранирует символы, скрипт не исполняется (XSS-защита). |
| 13 | Превышение максимальной длины поля (например, `НаимПорт` более 255 символов) | Ошибка: «Превышена допустимая длина поля (максимум 255 символов)». |
| 14 | Отправка запроса на смену статуса без электронной подписи (УКЭП) | Ошибка: «Документ должен быть подписан электронной подписью». |
| 15 | Использование просроченного сертификата электронной подписи | Ошибка: «Сертификат электронной подписи недействителен». |
| 16 | Попытка удалить или изменить завершённую ЭТрН (статус «Завершен») | Система запрещает редактирование, кнопки редактирования неактивны. |
| 17 | В поле «Номер ЭТрН» указать уже существующий номер (нарушение уникальности) | Ошибка: «Электронная транспортная накладная с таким номером уже существует». |
| 18 | Отправка файла без заполненного поля `УИД_ТрН` (или пустого) | Ошибка валидации: «Поле УИД_ТрН обязательно для заполнения». |
| 19 | При формировании XML указать неверный код `КНД` (например, `1110431` для грузоотправителя) | Система отклоняет файл с ошибкой: «Неверный код формы документа (КНД). Ожидается 1110430». |
| 20 | В поле «Дата погрузки» указать несуществующую дату (например, 32.01.2025) | Ошибка формата: «Дата должна соответствовать формату ДД.ММ.ГГГГ». |
| 21 | В поле «Время прибытия» указать неверное время (например, 25:70:00) | Ошибка формата: «Время должно соответствовать формату ЧЧ:ММ:СС». |
| 22 | В поле «Вес груза» указать число, превышающее максимально допустимую длину (19 цифр, например, 1000000000000000000) | Ошибка: «Значение превышает допустимый размер» или ошибка переполнения. |
| 23 | Попытка получить УИД от ГИС ЭПД, но сервис недоступен (таймаут) | Система выводит сообщение: «Сервис ГИС ЭПД временно недоступен. Попробуйте позже». |
