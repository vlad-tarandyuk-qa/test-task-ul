# Промпт для генерации чек-листа тестирования формы создания заявки на перевозку

Ты — эксперт по тестированию. Составь чек-лист (позитивные, негативные сценарии, граничные значения) для тестирования MVP формы создания заявки на перевозку.

## Контекст

Разрабатывается MVP формы создания заявки на перевозку для «УЛ Лайт». В форме есть следующие поля:

- **Город отправления**: автокомплит/выпадающий список из базы данных.
- **Город назначения**: автокомплит/выпадающий список из базы данных.
- **Вес груза**: числовое поле, в килограммах, максимум 20 000 кг.
- **Объем груза**: числовое поле, в кубических метрах, максимум 82 м³.
- **Дата погрузки**: датапикер, нельзя выбрать прошедшую дату.
- **Кнопка «Рассчитать стоимость»**: отправляет POST-запрос к API, возвращает цену в рублях или ошибку, если маршрут не найден.

Требуется:
1. Позитивные сценарии (happy path).
2. Негативные сценарии (ошибки валидации, недоступность API и т.д.).
3. Граничные значения (макс, мин, пустые поля, специальные символы).
4. Проверки интеграции с API и автокомплитом.

Оформи чек-лист в виде маркированного списка с группировкой по разделам.
