# Граничные условия для тестирования формы создания заявки на перевозку

Эти граничные условия выделены отдельно, потому что их проверка критична для стабильности MVP: они выявляют ошибки в валидации, логике расчёта и обработке исключений, которые могут привести к некорректным данным или сбоям. Особое внимание уделено комбинациям вес+объём, автокомплиту с учётом русскоязычных особенностей и обработке ошибок API.

## Вес груза (кг, максимум 20 000)
- **0** → ошибка «Значение должно быть больше 0»
- **1** → допустимо (минимальный положительный вес)
- **20 000** → допустимо (максимум)
- **20 001** → ошибка «Превышен максимальный вес»
- **дробные значения** (например, 10.5) → должны приниматься, если поле поддерживает десятичные дроби; если поле принимает только целые числа, то должна быть ошибка формата

## Объём груза (м³, максимум 82)
- **0** → ошибка «Объём должен быть больше 0»
- **0.1** → допустимо
- **82** → допустимо (максимум)
- **82.1** → ошибка превышения

## Комбинации вес + объём (критично для логистики)
- **Вес = 20 000 кг, объём = 0.1 м³** → нереалистичная плотность, требуется предупреждение: «Нереалистичная плотность груза, проверьте данные»
- **Вес = 1 кг, объём = 82 м³** → пустой фургон, логическое предупреждение: «Объём слишком велик для такого веса, проверьте данные»

## Дата погрузки
- **Вчера** → недоступна в датапикере
- **Сегодня** → доступна
- **Завтра** → доступна
- **Будущая дата (через 6 месяцев)** → доступна
- **29 февраля в невисокосный год** (например, 29.02.2027) → дата должна блокироваться или автоматически корректироваться на 28.02 или 01.03
- **Праздничные дни (1 января, 9 мая и др.)** → если бизнес-логика требует, должно быть предупреждение или блокировка; иначе — принимается как обычная дата (требует уточнения у заказчика)

## Автокомплит городов
- **Ввод 1–2 символов** → появляется список вариантов
- **Ввод с опечаткой** → список похожих или пусто
- **Несуществующий город** → «Город не найден»
- **Город с буквой «ё»** («Орёл») → находится корректно
- **Два города с одинаковым названием** → отображается регион или уточняющий атрибут
- **Вставка текста из буфера обмена (Ctrl+V)** → поле автокомплита корректно обрабатывает вставку

## API
- **Маршрут не найден** → ошибка 404, пользователь видит понятное сообщение: «Маршрут не найден»
- **Таймаут (>30 сек)** → сообщение «Сервис временно недоступен, попробуйте позже»
- **Ответ с невалидным JSON** → ошибка обработки, приложение не падает, выводится сообщение «Ошибка получения данных»
- **Пустой ответ (200, но нет поля price)** → сообщение «Ошибка расчёта стоимости»
- **Протухший токен** → код 401, перенаправление на страницу авторизации (а не техническая ошибка)
- **Изменение данных после расчёта** → старый результат должен быть скрыт или сброшен до нового запроса

## Кнопка «Рассчитать»
- **Двойной клик** → отправляется только один запрос, кнопка блокируется на время выполнения запроса
- **Клик при пустых обязательных полях** → запрос не отправляется, обязательные поля подсвечиваются с сообщением об ошибке

## Специальные символы и ввод
- **Ввод текста в числовое поле** («abc») → поле не принимает текст, либо выдаёт ошибку «Введите число»
- **Вставка из буфера (Ctrl+V)** → корректно обрабатывается, не приводит к сбою
- **Спецсимволы в числовых полях** (`<`, `>`, `&`, `"`, `;`) → игнорируются или выводят ошибку (не должны ломать форму)
- **Пробелы в числах** → обрезаются (trim) или выдаётся ошибка «Введите число без пробелов»
