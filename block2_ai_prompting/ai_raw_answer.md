# Чек-лист тестирования формы создания заявки на перевозку

## ✅ Позитивные сценарии

- [ ] Заполнить все поля корректными данными → нажать «Рассчитать стоимость» → получить цену в рублях
- [ ] Выбрать города с похожими названиями через автокомплит → убедиться, что выбран правильный вариант
- [ ] Ввести минимально допустимый вес (0,1 кг) и минимальный объём (0,1 м³) → получить стоимость
- [ ] Ввести максимально допустимый вес (20 000 кг) и объём (82 м³) → получить стоимость
- [ ] Выбрать дату погрузки — сегодняшнее число → успешный расчёт
- [ ] Выбрать дату погрузки — 6 месяцев вперёд → успешный расчёт
- [ ] Повторно нажать «Рассчитать стоимость» без изменения данных → получить тот же результат
- [ ] Изменить город отправления и пересчитать → новая корректная цена

---

## ❌ Негативные сценарии

### Валидация полей

- [ ] Нажать «Рассчитать» без заполнения ни одного поля → все обязательные поля подсвечены ошибкой
- [ ] Не выбрать город отправления → сообщение об ошибке под полем
- [ ] Не выбрать город назначения → сообщение об ошибке под полем
- [ ] Оставить поле «Вес» пустым → ошибка валидации
- [ ] Оставить поле «Объём» пустым → ошибка валидации
- [ ] Не выбрать дату погрузки → ошибка валидации
- [ ] Указать город отправления = город назначения → ошибка «Города совпадают»

### Некорректные значения

- [ ] Ввести вес = 0 → ошибка «Значение должно быть больше 0»
- [ ] Ввести вес = 20 001 кг (превышение максимума) → ошибка с указанием лимита
- [ ] Ввести объём = 0 → ошибка
- [ ] Ввести объём = 82,1 м³ → ошибка превышения
- [ ] Ввести отрицательный вес (-1) → ошибка
- [ ] Ввести отрицательный объём (-0,5) → ошибка
- [ ] Ввести в числовое поле текст («abc») → поле не принимает нечисловые символы или выдаёт ошибку
- [ ] Выбрать прошедшую дату в датапикере → дата недоступна для выбора
- [ ] Ввести дату вручную в прошлом (если поле допускает ввод) → ошибка валидации

### Ошибки API

- [ ] Маршрут не найден в базе → ответ API 404/ошибка → отображается понятное сообщение пользователю
- [ ] API недоступен (timeout) → показывается сообщение о недоступности сервиса, кнопка не висит бесконечно
- [ ] API возвращает 500 → отображается сообщение об ошибке сервера
- [ ] Потеря соединения во время запроса → пользователь видит ошибку сети
- [ ] Двойной клик по «Рассчитать» → отправляется только один запрос (кнопка блокируется)

## 🔢 Граничные значения

### Вес груза

- [ ] Вес = 0 → ошибка
- [ ] Вес = 0,001 → принимается или ошибка (уточнить минимум)
- [ ] Вес = 1 кг → успех
- [ ] Вес = 19 999 кг → успех
- [ ] Вес = 20 000 кг (максимум) → успех
- [ ] Вес = 20 000,1 кг → ошибка превышения
- [ ] Вес = 20 001 кг → ошибка

### Объём груза

- [ ] Объём = 0 → ошибка
- [ ] Объём = 0,1 м³ → успех
- [ ] Объём = 82 м³ (максимум) → успех
- [ ] Объём = 82,01 м³ → ошибка превышения
- [ ] Объём = 82,1 м³ → ошибка

### Дата погрузки

- [ ] Вчерашняя дата → недоступна в датапикере
- [ ] Сегодняшняя дата → доступна и принимается
- [ ] Завтрашняя дата → принимается
- [ ] Первый день следующего года → принимается
- [ ] 29 февраля в високосный год → обрабатывается корректно

### Специальные символы и данные

- [ ] Спецсимволы в числовых полях (`<`, `>`, `&`, `"`, `;`) → игнорируются или ошибка
- [ ] Пробелы в числовых полях → обрезаются (trim) или ошибка
- [ ] Очень большое число (1e20) → ошибка или корректная обработка
- [ ] Число с запятой вместо точки (1,5) → принимается или заменяется автоматически

## 🔍 Автокомплит и выпадающие списки

- [ ] Ввести 1–2 символа → появляется список вариантов из БД
- [ ] Ввести корректное название города → нужный город присутствует в результатах
- [ ] Ввести название с опечаткой → список показывает похожие варианты или пуст
- [ ] Ввести несуществующий город → список пуст, подсказка «Город не найден»
- [ ] Ввести название на разных регистрах («МОСКВА», «москва», «Москва») → одинаковый результат
- [ ] Выбрать город из списка → значение вставляется в поле корректно
- [ ] Очистить поле после выбора → состояние валидации сбрасывается
- [ ] Быстрый ввод (typing fast) → запросы к БД не дублируются (debounce)
- [ ] Поле автокомплита недоступно (БД не отвечает) → показывается сообщение об ошибке
- [ ] Навигация по списку клавишами ↑↓ и Enter → работает корректно

## 🔌 Интеграция с API

- [ ] Успешный запрос → цена отображается в рублях с разделителем тысяч
- [ ] Ответ API содержит нулевую цену (0 ₽) → отображается корректно, не скрывается
- [ ] POST-запрос содержит все ожидаемые поля: `originCity`, `destCity`, `weight`, `volume`, `loadingDate`
- [ ] Запрос не уходит при незаполненной форме (клиентская валидация блокирует)
- [ ] Ответ приходит с задержкой >3 сек → отображается индикатор загрузки
- [ ] Кнопка «Рассчитать» заблокирована во время ожидания ответа
- [ ] После получения ответа кнопка снова активна
- [ ] Проверить, что в запросе не передаются лишние/служебные поля
- [ ] Проверить корректность формата даты в запросе (ISO 8601 или требуемый формат API)
- [ ] При ошибке API повторный запрос после исправления данных работает корректно
